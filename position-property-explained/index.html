How it works

Static is the natural state of every element. Elements stack top-to-bottom, left-to-right according to the HTML order. There is no special positioning — the browser places each box right after the previous one.

Think of it like words in a paragraph: each word follows the last with no jumping around.

CSS code

.box {
  position: static; /* default — no need to write */

  /* these have NO effect with static: */
  top: 20px; /* ✗ ignored */
  left: 50px; /* ✗ ignored */
  z-index: 10; /* ✗ ignored */
}

How it works

The element stays in the normal flow (its original space is reserved) but visually shifts using top, right, bottom, or left.

Positive top pushes the element down. Positive left pushes it right. The offset is measured from where it would naturally sit.

It also becomes a positioning anchor — any absolutely positioned children will be placed relative to this element.

CSS code

.box {
  position: relative;

  /* offset from its natural position */
  top: 20px; /* shift ↓ 20px */
  left: 40px; /* shift → 40px */
}

/* Original space stays — siblings don't move */

The ancestor chain rule

An absolute element climbs up its ancestors until it finds one that is not static. That ancestor becomes the anchor. If none exists, it anchors to the page root.

<div style="position:relative"> ← ANCHOR ✓
<div style="position:absolute"> positions here

How it works

Absolute removes the element from the document flow completely. No ghost space is left behind — siblings collapse to fill the gap.

The element is then placed using top/right/bottom/left measured from its nearest positioned ancestor (any ancestor with a position other than static).

Common uses: tooltips, dropdown menus, badges, overlays, icons positioned within a card.

CSS code

/* Step 1: give the parent an anchor */
.parent {
  position: relative; /* ← required anchor */
}

/* Step 2: absolutely place the child */
.badge {
  position: absolute;
  top: 8px;
  right: 8px;
}

/* No ghost space — siblings fill the gap */

How it works

Fixed is like absolute but it ignores all ancestors. Its reference point is always the browser viewport — the visible rectangle of the window.

No matter how far the user scrolls, a fixed element stays anchored to the same viewport coordinates. It is also removed from the normal flow.

⚠️ Beware: if an ancestor has transform, filter, or will-change, it can break fixed positioning — the ancestor becomes a new containing block.

CSS code

.navbar {
  position: fixed;
  top: 0;
  left: 0;
  width: 100%; /* span full width */
  z-index: 1000; /* stay on top */
}

/* Compensate for the fixed bar height: */
body {
  padding-top: 60px;
}

🗓️ Each header flows normally until it reaches the top:0 threshold while scrolling — then it locks until its parent section leaves the viewport.

The two phases

Phase 1 — scrolling toward threshold
Behaves exactly like relative. In flow, taking up space, scrolling normally with the page.
Phase 2 — past the threshold
Behaves like fixed. Locks at the threshold point until its parent container is fully scrolled off.

How it works

Sticky is a hybrid of relative and fixed. It starts in the normal flow and scrolls with the page. Once it hits the threshold you define (e.g. top: 0), it locks in place — but only within the bounds of its parent container.

When the parent container scrolls off-screen, the sticky element is carried away with it. This is what lets multiple section headers replace each other as you scroll.

⚠️ Common gotcha: the parent must have a defined height and cannot have overflow: hidden, or sticky won't work.

CSS code

/* Table of contents, section headers, etc. */
.section-header {
  position: sticky;
  top: 0; /* stick when it reaches the top */
  z-index: 10;
}

/* ⚠️ Parent must NOT have overflow:hidden */
.section {
  overflow: visible; /* required */
}
